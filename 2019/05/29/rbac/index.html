权限管理系统-rbac

发表于 2019-05-29 | 分类于 综合
字数统计: 4.3k | 阅读时长 ≈ 16

需求分析

完成一个简单的RBAC权限管理系统,系统中包括部门管理,员工管理,权限管理,角色管理四个模块.

  • 基本的crud操作
  • 员工和角色的关系管理
  • 角色和权限的关系管理
  • 权限加载
  • 登录用户信息的检查
  • 权限检查和拦截

Mybatis

  • lazyLoadingEnabled:开启延迟加载

    • 如果没有开启延迟加载,在查询主对象的时候,也会立即查询和该对象关联的其他对象

  • aggressiveLazyLoading:积极加载

    • 开启积极加载.即使没有访问主对象关联的其他对象,也会执行响应的查询操作.浪费性能

  • lazyLoadTriggerMethods:触发延迟加载的方法

    • 默认情况下,访问主对象中的equals,clone,hashCode,toString方法时,会触发延迟加载立即查询关

      联对象,没有必要

表示哪些请求需要交给Spring MVC处理.使用:/定义默认servlet映射.使用例如*.do表示拦截处理所有以.do为后缀名的请求

元素标记容器是否在启动的时候就加载这个servlet

DepartmentServiceImpl

使用@Service注解,将被IoC注解析器扫描,自动注册到Spring容器.不需要在application.xml定义bean了.类似注解:@Component、@Repository、@Controller

使用@Resource和@Autowired的区别

相同:

  • 都可以用来装配bean. 都可以写在字段上,或写在setter方法上.

不同:

  • @Autowired默认按类型装配.默认情况下必须要求依赖对象必须存在,如果要允许null值,可以设置它的required属性为false.
    • 由spring提供
  • @Resource默认按照名称进行装配.名称可以通过name属性进行指定,如果没有指定name属性,.当注解写在字段上时,默认取字段名,按照名称查找.如果注解写在setter方法上默认取属性名进行装配.
    • 通过此注解完成从spring配置文件中查找名称为departmentMapper的bean来装配字段departmentMapper.
    • spring配置文件中不存在 departmentMapper名称的bean则转向按照bean类型进行查找
    • 由JDK1.6提供

封装PageResult

用户传入的两个字段:

  • currentPage(当前页)
  • pageSize(每页条数)

数据库查询出来的两个字段:

  • totalCount(总条数)
  • listData(每页的具体数据)

需要计算的三个值:

  • totalPage(总页数)
  • prevPage(上一页)
  • nextPage(下一页)

DepartmentMapper.xml

  • mapper.xml中的标签的id对应mapper接口中的方法名
  • 标签类型
    • parameterType->传入参数的类型(包装类型/对象)
    • useGeneratedKeys->默认为false.是否获取insert插入数据时主键自增的值.
    • keyProperty->把useGeneratedKeys的值赋给对象的指定(id)属性,使用setter获取.
    • keyColumn->当主键列不是表中第一列的时候,需要指定.
  • resultType
    • 返回的结果类型
      • 对象/数据类型/集合
      • 对象的属性必须和数据库的列名一一对应.
  • resultMap
    • 自定义返回的结果类型
      • column 数据表中的列名
      • property 对象中的属性(自定义和column对应的映射关系)
